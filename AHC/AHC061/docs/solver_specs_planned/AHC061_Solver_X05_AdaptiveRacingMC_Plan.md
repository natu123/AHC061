# AHC061 Solver X05 Adaptive Racing MC Plan

## 目的
- 固定サンプル数MCで発生した過保守化/過学習を避けつつ、`M=3..5` の `mean/median` を押し上げる。
- `x01` の高人数帯安定性を維持したまま、中人数帯の意思決定ノイズを低減する。

## 対象
- 系統ID（予定）: `x05`
- 系統名（予定）: `x05_adaptive_racing_mc`
- 状態: 計画中（未実装）

## 仕様（計画）
- 狙い:
  - 候補手ごとに同一シナリオ集合（common random numbers）で逐次評価し、優劣が確定した候補を早期打ち切りする。
- 別系統性:
  - `x03` の粒子事後分布 + CVaR最適化とも、`x04` の中期マクロ経路最適化とも異なり、1手選択の統計効率を改善する探索系。
- 期待効果:
  - `M=3..5` で `mean/median` を改善しつつ、`min` の悪化を抑える。
  - 固定サンプル増量で悪化した `T-062` の再発を防ぐ。

## 実装スコープ（初期）
- 候補生成:
  - `evaluate_local_move` で上位 `K` 候補を抽出（`K` は `M` と `phase` 依存）。
- 逐次評価:
  - 各ラウンドで全候補に同一の相手行動サンプルを適用し、`mean` と分散推定を更新。
  - 信頼区間（LCB/UCB）で劣後候補を除外し、残候補へ計算資源を再配分。
- 最終選択:
  - `mean - risk_w * std` を基本に、`downside_prob` ペナルティを加えたスコアで決定。
- 既存資産の再利用:
  - `build_ai_candidates_and_probs`, `sample_index`, `simulate_turn`, `strategic_score` を共通利用。

## 実験計画（初期）
- Small batch:
  - seed `0..19` で A/B（現champion `x01` vs `x05` 初版）
- Full eval条件:
  - `mean` 改善、かつ `min` 悪化が許容範囲内なら seed `0..99` へ拡張
- 記録必須:
  - mean/median/min/max, elapsed, 候補淘汰率（平均残候補数）

## 注意点
- 淘汰を早めすぎると探索不足で局所解に固定される。
- 逐次停止判定が強すぎると no-op に近い挙動へ退化する可能性がある。
