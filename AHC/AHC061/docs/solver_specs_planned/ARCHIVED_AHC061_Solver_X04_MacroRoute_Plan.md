# AHC061 Solver X04 Macro Route Plan

## 目的
- 1手評価中心から中期計画型へ移行し、育成と侵攻の連鎖効率を改善する。

## 対象
- 系統ID: `x04`
- 系統名: `x04_macro_route`
- 状態: 移管済み（2026-02-15）
- 参照先（built）: `docs/solver_specs_built/AHC061_Solver_X04_MacroRoute.md`
- 採用ログ: `docs/AHC061_Experiment_Log_2026-02.md` の `T-069`
- 起点commit: `437733e`
- 初期パラメータ（着手時）:
  - 計画長: `6..8`（`M`依存）
  - beam幅: `3..5`
  - 目標クラスタ数: `4..6`

## 仕様（計画）
- 狙い:
  - 6〜12ターンの「育成 -> 侵攻」シーケンスを直接最適化し、局所最適を回避する。
- 別系統性:
  - 現在の局所評価 + 短期rolloutから、中期のroute/sequence planningへ移行。
- 期待効果:
  - とくに`M=3..5`で高価値連鎖取得と育成効率の取りこぼしを減らし、`mean/median`改善を狙う。

## 実装スコープ（初期）
- マクロ行動候補:
  - 目標クラスタ（高価値未領土/敵Lv1帯/自領土育成帯）を抽出。
- プランナー:
  - 6〜12手の行動列を beam または限定探索で生成。
- 実行時:
  - 毎ターン再計画し、先頭手のみ出力。
- 既存資産の再利用:
  - `simulate_turn` と `evaluate_local_move` をプラン評価へ流用。

## 実験計画（初期）
- Small batch:
  - seed `0..19` で `M=3..5` を重点比較。
- Full eval条件:
  - `M=3..5`のmean改善が有意で、全体min悪化が軽微なら `0..99`へ。
- 記録必須:
  - mean/median/min/max, elapsed, `M`帯別サマリ。

## 注意点
- プラン長を伸ばしすぎると計算量が急増する。
- 相手行動の読み誤差が蓄積するため、再計画頻度と分岐幅のバランス調整が必要。
